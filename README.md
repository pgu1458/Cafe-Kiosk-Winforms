# ☕ 카페 키오스크 (Cafe Kiosk)

> C# WinForms 기반 카페 키오스크 시뮬레이터  
> 주문부터 결제, 회원 적립, 시선 제어까지 실제 서비스처럼 구현

🔗 [GitHub](https://github.com/pgu1458/cafe-kiosk-winforms)

---

## 프로젝트 소개

키오스크가 일상에서 워낙 자주 쓰이다 보니 직접 만들어보고 싶었습니다. UI 구성부터 결제 흐름, 다국어 처리, 시선 제어까지 최대한 실제 서비스처럼 구현하는 것을 목표로 했습니다. 단순한 UI 연습을 넘어서 실제 키오스크가 어떻게 동작하는지 흐름 전체를 직접 설계하고 구현했습니다.

## 주요 기능

### 🌐 시작 화면 — 언어 선택 및 BGM

<!-- 시작 화면 스크린샷 -->
![시작 화면](screenshots/start.png)

프로그램을 실행하면 가장 먼저 언어 선택 화면이 표시됩니다. 선택한 언어는 이후 모든 화면의 메뉴명과 UI 문구에 즉시 반영됩니다.

- 한국어 / 영어 / 일본어 / 중국어 4개 언어 지원
- 언어 선택에 따라 시작 문구 및 메뉴명 자동 변경
- 배경음악 재생 및 ON/OFF 토글 (AxWindowsMediaPlayer COM 컴포넌트 활용)

---

### 🛒 주문 화면

<!-- 주문 화면 스크린샷 (전체) -->
![주문 화면](screenshots/order.png)

**탭 구성 및 이용 유형 선택**

음료 / 디저트 / 굿즈 탭으로 메뉴가 분류되어 있습니다. 주문 전 매장 / 포장을 선택하며, 포장 선택 시 전체 금액에 **10% 할인이 자동 적용**됩니다.

**주문 목록 실시간 관리**

- 메뉴 버튼 클릭 시 주문 목록에 즉시 추가
- 항목별 개별 취소 및 전체 취소 기능
- 총액 실시간 계산 및 표시
- 선택한 언어로 메뉴명 자동 번역 표시

**실시간 시계**

화면 우측 상단에 현재 시간을 실시간으로 표시합니다.

### 👤 회원 기능 — 로그인 / 적립

<!-- 로그인 화면 스크린샷 -->
![로그인 화면](screenshots/login.png)

인메모리 딕셔너리 기반의 가상 회원 DB로 로그인과 회원가입을 구현했습니다. 실제 DB 없이도 앱 실행 중에는 회원 데이터가 유지됩니다.

- 로그인 / 회원가입 (인메모리 가상 DB)
- 로그인 시 쿠폰 1개 자동 적립
- 쿠폰 **10개 누적 시 무료 커피 1잔** 제공 안내
- 회원 로그인 시 전체 금액 **5% 할인** 자동 적용

**할인 중복 적용 예시**

| 조건 | 할인율 |
|------|--------|
| 포장 선택 | 10% |
| 회원 로그인 | 5% |
| 포장 + 회원 | 15% |

### 💳 결제

<!-- 결제 수단 선택 화면 스크린샷 -->
![결제 화면](screenshots/payment.png)

카카오페이 / 네이버페이 / 신용카드 / 현금 4가지 결제 수단을 지원합니다. 각 결제 수단 선택 시 해당 화면으로 전환되며, 결제 진행 중에는 로딩 애니메이션이 표시됩니다.

| 결제 수단 | 방식 |
|-----------|------|
| 카카오페이 | QR 코드 화면 표시 |
| 네이버페이 | QR 코드 화면 표시 |
| 신용카드 | 카드 삽입 안내 화면 |
| 현금 | 카운터 이동 안내 |

결제 완료 후 영수증 출력 여부를 선택할 수 있습니다.

### 👁️ 시선 제어 (Eye Tracking)

<!-- 시선 제어 동작 화면 스크린샷 -->
![시선 제어](screenshots/eye_tracking.png)

OpenCV(OpenCvSharp)와 Haar Cascade 분류기를 활용해 카메라로 눈 위치를 실시간 감지하고, 시선이 특정 버튼에 일정 시간 머물면 자동으로 클릭되는 기능입니다. 마우스나 터치 없이 시선만으로 키오스크를 조작할 수 있어 거동이 불편한 사용자의 접근성을 높이는 것을 목표로 구현했습니다.

- 카메라 입력 → Haar Cascade로 눈 영역 감지 → 시선 좌표 계산
- 시선이 버튼 영역에 일정 시간 유지되면 자동 클릭 트리거
- 화면 우측 ON/OFF 토글로 기능 활성화/비활성화 가능

> 조명 환경이나 카메라 품질에 따라 인식률 차이가 있을 수 있습니다.

## 실행 방법

1. 저장소 클론

```bash
git clone https://github.com/pgu1458/cafe-kiosk-winforms.git
```

2. Visual Studio에서 `Kiosk.csproj` 열기
3. NuGet 패키지 복원

```
OpenCvSharp4
OpenCvSharp4.runtime.win
```

4. `bin/Debug/net6.0-windows/` 경로에 아래 파일 확인
   - `click.wav`, `pay.wav`, `menu.wav`
   - `haarcascade_eye.xml`
5. 빌드 후 실행

## 한계 및 개선 방향

**데이터 영속성**  
현재 회원 데이터가 인메모리 딕셔너리에만 저장되어 앱 종료 시 초기화됩니다. SQLite 등 로컬 DB를 연동하면 데이터를 유지할 수 있습니다.

**시선 제어 인식률**  
Haar Cascade 기반 눈 인식은 조명 환경이나 카메라 품질에 따라 인식률 차이가 큽니다. MediaPipe Face Mesh 등 딥러닝 기반 모델로 교체하면 안정성을 높일 수 있습니다.

**결제 시뮬레이션**  
현재는 실제 결제 API 없이 UI 흐름만 구현된 상태입니다. 실제 서비스 적용을 위해서는 카카오페이·네이버페이 API 연동이 필요합니다.
